# 기능정책서 — 6인 회의 조율 서비스 (SyncFlow)

> 이 문서는 Claude Code가 그대로 읽어 화면을 구현하기 위한 화면별 정책서다.
> 구조적 근거: 인과사슬 ⓑ(선호 강도 미포착) → ⓒ(부담 비대칭). 모든 화면·기능은 이 인과사슬에 근거를 둔다(근거 없는 기능은 없음).
>
> ⚠️ 캐논: 이 파일은 사용자가 제공한 확정 스펙의 정본 복원본이다. 화면 구현 시 반드시 이 문서를 기준으로 한다.

---

## 0. 전역 정책 (모든 화면 공통)

| 정책 | 내용 |
|---|---|
| 진입 방식 | 대시보드·로그인 없음. 링크 기반 진입만 (주최자: 관리자 링크 / 참여자: 초대 링크) |
| 회의 스코프 | 단일 인스턴스(6인·1시간·다음주) 회의. 다중 회의 관리·미팅 히스토리 기능 없음 |
| 후보 시간 | 주최자가 구체적 후보 시간을 사전 지정하지 않음. 조율 기간(예: 다음주 평일)만 설정 |
| 프라이버시 | 가능/부담 여부는 항상 집계 수치로만 노출. 개인별 가능·사유는 어떤 화면에서도 타인에게 비노출 |
| 참여 상태 | "제출완료 / 대기중" 같은 참여 여부는 이름과 함께 노출 가능 (선호 강도와는 별개 정보) |
| 사전 텍스트 | 공개용 사유 입력 필드 없음. 단, 참여자 개인 전용 "메모 설정" 라벨은 본인에게만 노출되는 조건으로 허용 |
| 디자인 키 | 다크 배경, teal / (coral) 포인트 컬러, 카드형 레이아웃, 둥근 모서리, 넓은 여백 |
| 반응 기준 | 1920x1080 데스크탑 웹 기준 |

> 디자인 키 주의: 원문은 "다크 네이비 배경 + teal/coral"이나, 확정 디자인 시스템은 **다크 그레이(grey-900) + teal/orange** 다. 색은 디자인 시스템 토큰을 사용한다(coral≈orange로 대응). 불일치는 확인 후 조정.

## 1. Screen — 회의 생성 (역할: 주최자, 최초 진입 화면)

**목적:** ⓒ(부담 비대칭)이 발생하지 않도록, 주최자가 후보 시간을 미리 정하지 못하게 만든 화면.

**구성 요소**
- 회의 제목 입력 (text input)
- 참여자 등록 (이름 또는 이메일, 추가/삭제 가능, 태그 형태로 표시)
- 소요시간: `1시간` 고정값, 비활성(disabled)으로 표시
- 조율 기간: 기본값 "다음주 평일", 변경 가능 (달력 피커)
- 마감일 선택 (날짜 + 시간)
- CTA 버튼: "초대 보내기"

**금지 요소**
- 구체적 후보 시간(예: 화요일 10시) 입력 필드 → 절대 추가하지 말 것

**검증 규칙**
- 회의 제목 미입력 시 CTA 비활성
- 참여자 0명 상태에서 CTA 비활성
- 마감일이 조율 기간보다 이전일 경우 에러 메시지 표시

**전송 후 동작**
- 참여자 전원에게 초대 링크 발송 (이메일/알림 목적)
- 주최자에게 관리자 링크 발급 (현황보드로 재진입 가능한 고정 URL)
- "현황보드"로 자동 이동

---

## 2. Screen — 가능 입력 (역할: 참여자, 초대 링크 진입 시 최초 화면)

**목적:** ⓑ(선호 강도)를 이분법이 아닌 4단계로 입력받는 핵심 화면.

**구성 요소**
- 상단: 회의 제목, 주최자 이름, 소요시간, 조율 기간, 마감 D-day 표시
- 4단계 강도 범례 (세그먼트 컨트롤): `불가 / 부담 / 괜찮음 / 좋음`
- 그리드: 세로축 = 시간대(예: 09:00~18:00, 1시간 단위), 가로축 = 요일(조율 기간 내 평일)
- 그리드 셀 인터랙션: 클릭 또는 드래그로 선택한 강도를 여러 셀에 한 번에 적용(칠하기)
- "+ 메모 설정 추가" 버튼 → 3번 모달 실행
- CTA 버튼: "저장"

**검증 규칙**
- 전체 셀 중 최소 1개 이상 강도가 지정되지 않으면 "저장" 클릭 시 미입력 안내 노출, 저장 차단
- 미입력 슬롯은 그리드 영역을 강조 표시(예: 테두리 강조) 후 해당 위치로 스크롤

**저장 후 동작**
- "응답 요약" 화면으로 이동

---

## 3. Modal — 메모 설정 (역할: 참여자 개인 전용, "+ 메모 설정 추가" 클릭 시)

**목적:** 반복되는 개인 패턴(예: "매주 화~목 점심 직후는 항상 부담")을 그리드에서 하나하나 고르지 않고 일괄 적용.

**구성 요소 (4단계 입력)**

| 순서 | 항목 | 입력 방식 | 세부 |
|---|---|---|---|
| 1 | 패턴 선택 | 칩 선택 (단일) | `불가 패턴` / `가능 패턴` 중 하나 |
| 2 | 강도 선택 | 라디오 (패턴 선택에 종속) | `불가 패턴` 선택 시 → `불가`/`부담` 중 선택. `가능 패턴` 선택 시 → `괜찮음`/`좋음` 중 선택 |
| 3 | 적용 시간대 | 시작~종료 타임피커 | 예: 14:00 ~ 15:00 |
| 4 | 반복 여부 | 토글 (요일별) | 월~금 중 다중 선택 가능. 전부 미선택 시 "1회성"으로 처리(현재 조율 기간 내 해당 요일 1개에만 적용) |

**추가 필드 (선택)**
- 라벨명: 자유 텍스트 (예: "회의 집중도 선호") — **본인 화면에서만 노출, 현황보드·확정 화면·타 참여자에게 절대 비노출**

**동작**
- "적용" 클릭 시: 지정한 시간대 × 반복 요일에 해당하는 그리드 셀에 2번에서 고른 강도를 일괄 적용
- 기존에 이미 값이 있는 셀은 새 설정 값으로 덮어씀 (경고 없이 즉시 반영, 그리드로 돌아가면 결과 확인 가능)
- 모달을 여러 번 열어 여러 개의 규칙 생성 가능

> 🔄 갱신(2026-07-12, 사용자 지시): 이 모달은 "메모"가 아니라 **회의 선택 조건 커스터마이징**이다. "조건 저장" 시 규칙이 사라지지 않고 **조건 칩**으로 영속화되어 화면에 남는다. 각 칩은 **그리드(히트맵) 자동선택 컨트롤 버튼**으로, 클릭하면 해당 조건을 그리드에 다시 자동 적용한다(× 로 삭제). 최초 저장 시 1회 즉시 적용도 함께 이뤄진다. (기존 "규칙을 리스트로 저장하지 않음" 정책을 이 지시로 대체.)

**검증 규칙**
- 패턴 선택 미선택 시 "적용" 버튼 비활성
- 시작 시간이 종료 시간보다 늦을 경우 에러 표시
- 조율 기간에 포함되지 않는 요일은 반복 토글에서 선택 불가(비활성 표시)

## 4. Screen — 응답 요약 (역할: 참여자, 저장 후)

**목적:** 제출 확인 및 마감 전 수정 창구.

**구성 요소**
- 제출 완료 메시지
- (선택) 응답 통계 요약
- "수정하기" 버튼 → 가능 입력 화면(2번)으로 복귀, 재제출 시 기존 응답 덮어씀

---

## 5. Screen — 현황보드 (역할: 주최자)

**목적:** ⓒ(부담 비대칭)을 가시화하고, 주최자가 부담이 적은 시간을 선택하도록 지원.

**구성 요소**
- 응답 진행률: "n / 전체" (progress bar)
- 전체 히트맵: 참여 가능한 시간 자동 집계, 셀 클릭 시 해당 슬롯 상세(참여 n명, 부담 n명 → 집계 수치만) 팝업 표시
- 추천 후보 카드 2개 (Top 2): 요일·시간, 참여 가능(예: 7/8), 부담 인원 수(예: 1명) → **이름 비노출**
- 추천 후보 카드 선택 시 "이 시간으로 확정" 버튼 활성화
- CTA 버튼: "이 시간으로 확정" (클릭 즉시 처리, 별도 확인 모달 없음)

**예외 상태**
| 상태 | 처리 |
|---|---|
| 마감일까지 미응답자 존재 | 진행률에 미응답 인원 표시, 조기 확정 여부는 주최자 판단(버튼 비활성화하지 않음) |
| 추천 Top 2 모두 부담 인원 1명 이상 | 카드 상단에 "부담 최소 후보" 배지 표시, 주최자가 직접 판단해 확정 진행 |
| 응답 0명 | 히트맵 빈 상태(empty state) 표시, "아직 응답이 없어요" 안내 + "리마인드 보내기" 버튼 |

---

## 6. Screen — 확정 완료 (역할: 공통, 확정 버튼 클릭 후)

**목적:** 조율 종료 고지.

**구성 요소**
- 확정된 요일·시간 (큰 타이포)
- "전원에게 알림이 발송되었어요" 메시지
- 참여자 수, 확정 일정 요약 정보

**금지 요소**
- 별도 확인/재확인 모달 없음 → 현황보드에서 CTA 클릭 시 바로 이 화면으로 전환

---

## 7. 화면 전환 요약

```
[주최자] 회의 생성 → (초대 발송) → 현황보드 → 확정 완료
[참여자] (초대 진입) → 가능 입력 → (선택: 메모 설정 모달) → 저장 → 응답 요약 → (수정 시 가능 입력으로 복귀)
```

## 8. 구현 우선순위 (Claude Code 작업 시작점)

1. 가능 입력 화면 (그리드 + 4단계 세그먼트 + 드래그 칠하기) — 가장 공이 큰 핵심 화면
2. 현황보드 (히트맵 + 추천 카드 로직)
3. 회의 생성 / 확정 완료 / 응답 요약 (상대적으로 단순한 폼·안내 화면)
4. 메모 설정 모달 (부가 기능, 그리드 입력 완성 후 진행)
