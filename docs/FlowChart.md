# Flow Chart — 최종본 (SyncFlow)

> 화면 순서는 인과사슬(ⓑ → ⓒ)에서 도출된다. 와이어프레임은 개별 화면의 시각적 톤(4단계 세그먼트, 히트맵, 응답 요약 레이아웃)만 참고하고, 화면 간 연결 순서·모달 유무는 와이어프레임을 따르지 않았다.
>
> ⚠️ 캐논: 확정 스펙 정본 복원본. [[기능정책서]], [[IA]]와 함께 화면 구현 기준.

---

## Flow 1 — 참여자: 선호 강도를 입력·수정하는 흐름 (core)

```
[초대 알림 수신]
       │
       ▼
[Screen 1-1] 진입
  회의 제목 · 소요시간 1h · 조율 기간 확인
       │
       ▼
[Screen 1-2] 가능 입력  ← 핵심 화면
  시간대 × 요일 그리드에 4단계 강도 지정 (불가/부담/괜찮음/좋음)
  (선택) [Modal] 메모 설정 → 패턴 선택(불가/가능) · 강도 선택 · 적용 시간대 · 반복 여부
  ※ 라벨은 작성자 본인에게만 노출, 주최자 사전 지정 후보 없음
       │
       ▼
[Screen 1-3] 응답 요약
  제출 확인 · 응답 통계(선택)
       │
       ├─ 마감 전이면 [수정하기] → Screen 1-2로 복귀
       │
       ▼ (마감 도달 또는 수정 없음)
     대기 상태 유지 (주최자 확정까지)
```

**예외 처리**
| 상황 | 처리 |
|---|---|
| 그리드 일부 셀 미입력 상태로 제출 시도 | 미입력 슬롯 안내 → Screen 1-2로 복귀 유도 |
| 마감 전 응답 수정 | Screen 1-3 → Screen 1-2 복귀, 재제출 시 기존 응답 덮어씀 |

## Flow 2 — 주최자: 현황을 보고 확정하는 흐름 (downstream)

```
[Flow 2 진입]
       │
       ▼
[Screen 2-1] 회의 생성
  회의 제목 · 참여자 등록 · 소요시간 1h(고정)
  조율 기간(기본값 제공, 조정 가능) · 마감일
  ※ 구체적 후보 시간 지정 없음
       │
       ▼
[초대 발송] ───────────────► Flow 1 (참여자, 비동기 진행)
       │                         │
       │                         ▼
       │                  [응답 데이터 실시간 집계]
       ▼
[Screen 2-2] 현황보드  ← 핵심 화면
  응답 진행률(n/전체)
  히트맵(참여 가능 자동 집계, 개인 비식별)
  추천 후보 Top 2(참여 가능 + 부담 인원 수, 이름 비노출)
       │
       ▼
  [확정] 버튼 클릭 (별도 확인 모달 없음)
       │
       ▼
[Screen 2-3] 확정 완료
  확정 시간 고지 · 전원 알림 발송
```

**예외 처리**
| 상황 | 처리 |
|---|---|
| 마감일까지 미응답자 존재 | 현황보드에 미응답 인원 표시, 조기 확정 여부는 주최자 판단 |
| 추천 Top 2 모두 부담 인원 존재 | "부담 최소 후보" 안내, 주최자가 직접 판단해 확정 |

## 통합 흐름

```
주최자                          참여자 (n명)
  │
  ├─ 회의 생성 (후보 미지정) ───► 초대 진입
  │                              │
  │                              ├─ 가능 입력 (비동기, 각자)
  │                              ├─ 응답 요약 · (수정 가능)
  │                              └─ ...
  │                              │
  └────◄ 응답 집계 실시간 반영 ───┘
  │
  ├─ 현황보드 확인 (히트맵·추천, 집계만)
  ├─ 확정 (버튼 클릭)
  │
  └─ 확정 알림 발송 ───────────► 전원 수신
```
