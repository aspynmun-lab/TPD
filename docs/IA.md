# IA (Information Architecture) — 최종본 (SyncFlow)

> **구조적 근거는 리서치에서 확립한 인과사슬(ⓑ 선호 강도 미포착 → ⓒ 부담 비대칭)이다.**
> 와이어프레임(SyncFlow)은 화면 간 연결 순서의 근거가 아니라, 개별 화면이 어떤 느낌으로 보일지에 대한 시각 참고 자료로만 반영한다. 와이어프레임에 있는 화면이라도 인과사슬 근거가 없으면 구조에서 제외한다(예: 확정 전 별도 확인 모달 → 삭제).
>
> ⚠️ 캐논: 확정 스펙 정본 복원본. [[기능정책서]], [[FlowChart]]와 함께 화면 구현 기준. 서비스명 = SyncFlow.

---

## 0-1. 지향적 구조 — 대시보드 없음

이 서비스는 "여러 회의를 관리하는 상시 도구"가 아니라 **한 번에 6인 회의를 조율하는 단일 인스턴스**다. 따라서 "예정된 회의 목록", "종료된 회의" 같은 다중 회의 관리 기능은 인과사슬(ⓑ→ⓒ)에 근거가 없어 IA에서 제외한다. 진입은 로그인·대시보드가 아니라 **링크 기반**이다.

| 역할 | 진입 방식 |
|---|---|
| 주최자 (최초) | "회의 생성" 화면이 곧 첫 화면 |
| 주최자 (재진입) | 생성 시 발급된 관리자 링크로 현황보드 직접 진입 |
| 참여자 | 초대 링크 클릭 → 가능 입력 화면 직접 진입 (목록 없음) |

**제외 사유:** 여러 회의를 나열하는 화면이 생기는 순간 계정 시스템·미팅 히스토리·팀 관리 같은 부가 기능이 따라붙게 되고, 이는 Togedule/MeetMate 전체 스펙을 배끼는 것과 같은 이유로 스코프 밖이다.

## 0. 역할 구분

| 역할 | 진입 | 인과사슬 위치 |
|---|---|---|
| 참여자 | Flow 1 | ⓑ 선호 강도 발현 지점 |
| 주최자 | Flow 2 | ⓒ 부담 비대칭 가시화·해소 지점 |

## 1. 최상위 구조

```
Root (대시보드 없음 — 링크 기반 진입)
├── 1. 회의 생성 (주최자 전용, 최초 진입 화면)
│   ├── 회의 제목
│   ├── 참여자 등록
│   ├── 소요시간 1h (고정값)
│   ├── 조율 기간 (기본값 제공, 조정 가능)
│   ├── 마감일
│   └ ※ 구체적 후보 시간 사전 지정 없음
│
├── 2. 가능 입력 (참여자 전용) ── 핵심 플로우
│   ├── 그리드 (시간대 × 요일)
│   ├── 4단계 강도 (불가 / 부담 / 괜찮음 / 좋음)
│   ├── 메모 설정 (선택, 개인 전용) ── 모달
│   │   ├── 패턴 선택: 불가 패턴 / 가능 패턴 (칩 선택)
│   │   ├── 강도 선택: 패턴 내 세부 단계
│   │   ├── 적용 시간대: 시작~종료
│   │   ├── 반복 여부: 요일별 반복 또는 1회성
│   │   └── 라벨은 본인만 확인, 타인에게 비노출
│   └── 제출
│
├── 3. 응답 요약 (참여자 전용)
│   ├── 제출 확인
│   └── 마감 전 수정 가능 → 2번으로 복귀
│
├── 4. 현황보드 (주최자 전용) ── 핵심 플로우
│   ├── 응답 진행률 (n/전체)
│   ├── 히트맵 (참여 가능 자동 집계)
│   ├── 추천 후보 (참여 가능 + 부담 인원 수, 이름 비노출)
│   └── 확정 액션 (버튼 클릭 즉시 처리, 별도 확인 모달 없음)
│
└── 5. 확정 완료 (공통)
    ├── 확정된 시간 고지
    └── 전원 알림 발송
```

## 2. 화면별 데이터 요소

### 2.1 가능 입력 (참여자)

| 요소 | 표시 | 비고 |
|---|---|---|
| 시간대 × 요일 셀 | 4단계 세그먼트 | 주최자 사전 지정 후보 없음 |
| 메모 설정 | 개인 전용 반복 규칙 | 라벨(예: "회의 집중도 선호")은 본인 화면에서만 노출. 현황보드·확정 화면에 절대 노출 금지 |
| 사전 태그(공개용) | 없음 | 강도 척도와 기능 중복 + 프라이버시 역행으로 제외 |

### 2.2 현황보드 (주최자)

| 요소 | 표시 | 비고 |
|---|---|---|
| 히트맵 셀 | 집계 수치 | 개인 식별 불가 |
| 부담 인원 수 | 카운트만 | 이름·사유 비노출 → 와이어프레임 page7 방식(이름+사유 노출) 채택하지 않음 |
| 추천 후보 | Top 2 | 참여 가능 수 + 부담 최소 기준 |

## 3. 채택 / 제외 근거

| 항목 | 출처 | 판단 | 사유 |
|---|---|---|---|
| 4단계 강도 그리드 | 와이어프레임 | 채택 | groupTime 근거와 일치 |
| 메모 설정(개인 반복 규칙) | 와이어프레임 | 채택(조건부) | 타인 비노출 조건 하에 입력 효율 도구로 허용 |
| 응답 요약 + 마감 전 수정 | 와이어프레임 | 채택 | 참여자 쪽 자연스러운 종료 지점 |
| 메모 설정 모달(패턴·강도·시간대·반복) | 이번 결정 | 채택 | 반복 개인 패턴을 그리드 수동 입력 없이 일괄 적용, 라벨은 본인만 노출 |
| 히트맵 부담 인원 집계(이름 비노출) | 와이어프레임(page4) | 채택 | DR5 프라이버시 관련 해결책과 일치 |
| 이름 + 개인 사유 노출 (Confirm Meeting Slot) | 와이어프레임(page7) | 제외 | page4와 모순, 프라이버시 역행 |
| 확정 전 별도 확인 모달 | 와이어프레임(page7 구조) | 제외 | 인과사슬 근거 없음 → 확정 버튼 클릭으로 충분 |
| 주최자의 구체 후보 시간 사전 지정 | 없음(과거 이슈) | 제외 | Background에서 지목한 문제 재현 |
